TECHNICAL DECISIONS
================================================================================

1. Single Modal for All Features
   ✅ Reduces code duplication
   ✅ Consistent user experience
   ✅ Easier to maintain
   ✅ Conditional field visibility for clarity

2. Levenshtein Distance for Similarity
   ✅ Proven algorithm
   ✅ No external library needed
   ✅ Works for any string
   ✅ Configurable threshold

3. Weighted Duplicate Scoring
   ✅ 50% description — most important
   ✅ 40% amount — secondary matching
   ✅ 10% date — tertiary check
   ✅ 70% threshold — balanced false positive/negative

4. localStorage for Templates
   ✅ No backend needed
   ✅ Fast access
   ✅ Simple JSON serialization
   ✅ Scalable for hundreds of templates

5. Metadata-Only Attachments
   ✅ Keeps data size small
   ✅ Ready for cloud integration
   ✅ No base64 bloat in localStorage
   ✅ Production: integrate with file service
